Parallel Coordinates, Parallel Sets and Time Series

In the past lectures, you have been introduced to parallel coordinates and parallel sets, as well as different techniques for visualizing time series data. In this assignment, you're going to visualize weather data using these types of visualizations and connect them using interaction techniques.

The weather and stations data is loaded already from the corresponding files in the index.js, and printed to the console. Look carefully at the available data and analyze its attributes to make the best decisions for your design that will allow you to answer the given questions.

Task: Create a coordinated multi-view to analyze the weather data.

It is your task to design and implement the visualization system that allows the user to answer the following questions:

  1. In which year and in which weather station has the lowest temperature been recorded?
  2. Are there time-based patterns regarding humidity?
  3. Is there some kind of relation between temperature and humidity?
  4. a) Is air pressure lower by the sea or at continental locations?
    b) Is there a difference in air pressure between sea-level stations and mountain-based stations?
Add any elements that may be necessary in this HTML, in the #visualization div on top.

System requirements

Your visualization system should have the following features:
  1. A parallel coordinate system (20%).
  2. A parallel sets plot (20%).
  3. A suitable time visualization (20%).
  4. Connect the visualizations with brushing and linking (10%).
  5. Filtering items by attribute (10%).
